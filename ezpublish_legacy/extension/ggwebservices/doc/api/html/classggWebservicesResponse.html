ggWebservicesResponse::decodeStream (   $request,
  $stream,
  $headers = false,
  $cookies = array(),
  $statuscode = "200" 
)
abstract

Decodes the response from a text stream (usually the http response body).

Sets internal members: Value, IsFault, FaultString and FaultCode Should also set: StatusCode, Charset, ContentType, Headers, Cookies -

See Also
decodeStreamCommon for an easy way to do that Otoh it does not have to care about http status code, as handling http is left to client class Name is not set to response from request - a bit weird... but client injects name using the request one anyway The API is a bit whacky because we want too keep compat with the eZP original version
Parameters
ggWebservicesRequest$requestthe original request object
string$streamthe complete HTTP response in case $headers is false, the response body if $headers is an array
array$headersthe http headers received along with the response
string$statuscodehttp status code
Returns
void (if parsing of stream fails, set isFault to true plus error code and description)
