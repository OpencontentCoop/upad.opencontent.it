ggRESTRequest::decodeStream (   $rawRequest)

Unlike other requests, $rawRequest here is not used, as GET params are used, and for POST we rely on $_POST as it is already there for us.

This is a small break of the encapsulation principle of the API, but is faster than having to push this into a specific server class. While at it, we examine request headers also to determine response type for later. Q: what if req. body is not multipart/form-data? A: we just ignore it...

recover method name from the last fragment in the URL
